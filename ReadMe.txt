2 	Для поиска наилучшей модели:
Для задачи важна метрика recall, а precision не так важен. Accuracy не показателен, поэтому было определено посмотреть F-beta, которая позволяет сбалансировать внимание к одной из метрик. А так как нам важнее recall - соответственно с beta>1. При этом метрика F-beta зависит от порога классификации (который у нас по умолчанию 0.5) - чтобы оценить модель вне зависимости от порога - было опредлено ориентироваться по roc-auc; А уже после выбора модели по наилучшей roc_auc был подобран порог классификации, где f_beta получилась наилучшей;
целевая переменная была закодирована при помощит LabelEncoder;
были построены 2 пайплайна - один без сэмплирования, другой с сэмплированием;
была выбрана наилучшая модель с помощью RandomizedSearchCV, в котором применялись:
OHE - энкодер - для категориальных признаков с небольшим количеством категорий;
Ordinal - энкодер для категориальных признаков с большим количеством категорий;
из скейлеров был произведен перебор: StandardScaler(), MinMaxScaler() и RobustScaler(). Наилучшим оказался StandardScaler();
была произведена оценка roc-auc для моделей:
KNN c подбором гиперпараметра n_neighbors;
DecisionTreeClassifier с подбором гиперпараметров max_depth и max_features;
LogisticRegression (solver='liblinear', penalty='l1') с подбором параметра регуляризации С;
SVC с подбором гиперпараметров C, kernel, degree и gamma
